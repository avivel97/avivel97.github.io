Деплой

Vega — визуализации

1. Графики строятся в браузере

Вся работа происходит на устройстве пользователя.
→ Минимальная нагрузка на сервер
→ Легко добавлять пользователей

2. Нет состояния на сервере

Каждый запрос независим — система не хранит сессии.
→ Стабильность при пиках нагрузки
→ Простое масштабирование

3. Декларативный подход

Интерактивность внутри Vega, не через callback'и.
→ Предсказуемая работа
→ Меньше ошибок

DuckDB + File Storage

1. Hot–cold разделение данных

DuckDB для горячих данных; холодные — в файлах.
→ Быстрая колоночная in-memory БД для работы сессии
→ Безлимитная история в дешёвом хранилище

2. Загрузка по требованию

В DuckDB только нужное сейчас, остальное - по нажатию кнопки.
→ Каждый пользователь грузит нужное ему.
→ Вычисления на BIG DATA - вне БД, результаты в DuckDB

3. Предсказуемая производительность

Дашборды работают с датасетом стабильного размера.
→ Устойчивая latency под нагрузкой
→ Простой деплой, лёгкое кеширование

→ Каждый график запрашивает свои данные
→ Бэкенд возвращает готовые агрегаты
→ Загружается только нужное для визуализации

У RetailBI объём подгружаемых данных минимален — каждый график получает только рассчитанные значения в нужной детализации.
